# Phase 5 — 도전 과제

## #24 RoPE / ALiBi

- **절대 vs 상대 위치**: sinusoidal/learned는 "절대 위치"를 더한다. 하지만 언어에서 중요한 건
  주로 "상대적 거리"(두 단어가 얼마나 떨어졌나). 상대위치 방식은 이걸 직접 모델링.
- **RoPE(Rotary PE)**: Q,K 벡터를 위치각도만큼 **회전**시킨다. 두 토큰의 내적이 자연스럽게
  상대 거리에 의존하게 된다. 곱셈적(회전)이라 임베딩에 더하지 않고 attention 계산 안에서
  적용. LLaMA 등 다수가 채택.
- **ALiBi**: attention 점수에 거리 비례 **선형 페널티**를 더한다(멀수록 감점). 구현이 단순하고,
  학습보다 긴 길이로 **외삽(length extrapolation)**이 잘 되는 게 강점.

## #25 데모 공유

- **추론 배포의 최소 형태**: 학습된 가중치를 로드해 입력을 받아 생성 결과를 내주는 얇은 래퍼.
  학습 코드와 추론 코드를 분리하는 감각.
- **가중치 관리**: `state_dict` 저장/로드, 체크포인트 버저닝.
- **재현성 문서화**: "이 가중치 = 이 코드 + 이 데이터 + 이 seed"를 명시.
