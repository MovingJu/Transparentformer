# Phase 3 — 실제로 학습시키기

## #17 Toy seq2seq 학습

- **왜 toy task부터**: 실데이터로 바로 가면 "모델 버그"인지 "데이터/하이퍼파라미터 문제"인지
  구분이 안 된다. 복사(copy)처럼 정답이 자명한 과제는 **모델이 맞으면 반드시 100%에 수렴** —
  안 되면 100% 내 코드 버그. 최고의 디버깅 렌즈.
- **teacher forcing**: 디코더 입력 = 정답을 한 칸 오른쪽으로 민 것(`<bos>` + tgt[:-1]), 디코더가
  예측할 정답 = tgt. 학습은 이 "다음 토큰 맞히기"를 병렬로 한 번에.
- **손실 함수**: `CrossEntropyLoss`를 `(B*T, vocab)` vs `(B*T,)`로 펴서 계산. pad 위치는
  `ignore_index`로 제외.
- **과적합 sanity check**: 데이터 몇 개만으로 일부러 overfit 시켜 loss가 0 근처까지 가는지 —
  "학습 자체가 되는가"를 가장 먼저 확인하는 표준 기법.

## #18 Decoder-only 미니 GPT

- **decoder-only 구조**: 인코더·cross-attention을 떼고, causal self-attention 블록만 쌓는다.
  입력=출력이 같은 시퀀스이고, "다음 토큰 예측"만 한다. GPT 계열의 뼈대.
- **character-level LM**: 토크나이저 없이 문자 하나를 토큰으로. vocab이 작아(수십 개) 학습이
  빠르고, 토크나이징 복잡도 없이 언어모델 원리에 집중.
- **autoregressive 생성**: 시작 토큰에서 출발해 ① 다음 토큰 확률 예측 → ② 하나 뽑기 → ③ 뒤에
  붙여 다시 입력 → 반복. 학습은 병렬(teacher forcing)이지만 생성은 순차라는 비대칭.

## #19 생성 디코딩

- **greedy**: 매 스텝 확률 최댓값 토큰을 고름. 결정론적(같은 입력→같은 출력)이지만 단조롭고
  반복에 잘 빠진다.
- **temperature**: 로짓을 `logits / T`로 나눠 softmax. `T<1`이면 분포가 뾰족해져(보수적),
  `T>1`이면 평평해져(다양·모험적). `T→0`은 greedy 수렴. "창의성 다이얼".
- **top-k**: 확률 상위 k개만 남기고 나머지는 0으로 만든 뒤 그 안에서 샘플. 말도 안 되는 저확률
  토큰이 뽑히는 사고를 막는다.
- **top-p(nucleus)**: 누적확률이 p가 될 때까지의 토큰만 후보로. 분포가 뾰족/평평함에 따라 후보
  수가 적응적으로 변한다.
